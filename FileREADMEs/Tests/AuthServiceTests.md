# `AuthServiceTests.cs` — Test File

**Location:** `LearnSphereBackend.Tests/AuthServiceTests.cs`

---

## What This File Does

Contains **unit tests** for `AuthService`. Tests use the NUnit testing framework and Moq to mock dependencies. Each test verifies a specific behavior of the authentication service in complete isolation from the database.

---

## Test Setup

```csharp
public class AuthServiceTests
{
    private readonly Mock<IUserRepository>    _mockUsers;
    private readonly Mock<IStudentRepository> _mockStudents;
    private readonly Mock<IJwtTokenService>   _mockJwt;
    private readonly AuthService              _service;

    public AuthServiceTests()
    {
        // Create mocks for all dependencies
        _mockUsers    = new Mock<IUserRepository>();
        _mockStudents = new Mock<IStudentRepository>();
        _mockJwt      = new Mock<IJwtTokenService>();

        // Inject mocks into the real AuthService
        _service = new AuthService(_mockUsers.Object, _mockStudents.Object, _mockJwt.Object);
    }
}
```

`_mockUsers.Object` is a fake implementation of `IUserRepository` generated by Moq. The real `UserRepository` (which talks to SQL Server) is never used.

## Example Test: Valid Login

```csharp
[Fact]
public async Task LoginAsync_ValidCredentials_ReturnsAuthResponse()
{
    // ARRANGE — Set up fake data and fake behavior
    var user = new User { Name = "Test", Email = "test@example.com", Status = "active" };
    user.PasswordHash = hasher.HashPassword(user, "secret"); // Real hash (not mocked)

    _mockUsers.Setup(r => r.GetByEmailAsync(It.IsAny<string>(), It.IsAny<CancellationToken>()))
              .ReturnsAsync(user);   // When GetByEmailAsync is called → return fake user

    _mockJwt.Setup(j => j.CreateToken(It.IsAny<User>()))
            .Returns("fake-jwt-token"); // When CreateToken is called → return fake token

    // ACT — Call the real method
    var result = await _service.LoginAsync(
        new LoginRequestDto { Email = "test@example.com", Password = "secret" },
        CancellationToken.None
    );

    // ASSERT — Verify the output
    Assert.NotNull(result);
    Assert.Equal("Test",           result.Name);
    Assert.Equal("fake-jwt-token", result.Token);
}
```

## Example Test: Wrong Password

```csharp
[Fact]
public async Task LoginAsync_WrongPassword_ThrowsUnauthorized()
{
    var user = new User { Status = "active" };
    user.PasswordHash = hasher.HashPassword(user, "correctpassword");

    _mockUsers.Setup(r => r.GetByEmailAsync(It.IsAny<string>(), default))
              .ReturnsAsync(user);

    // ACT + ASSERT
    await Assert.ThrowsAsync<UnauthorizedAccessException>(
        () => _service.LoginAsync(
            new LoginRequestDto { Email = "x@x.com", Password = "wrongpassword" },
            CancellationToken.None
        )
    );
}
```

## Arrange-Act-Assert (AAA) Pattern

| Phase       | What It Does                                                    |
| ----------- | --------------------------------------------------------------- |
| **Arrange** | Set up mock behaviors, create test data, configure expectations |
| **Act**     | Call the actual method being tested                             |
| **Assert**  | Verify the output matches expectations                          |

Every test follows this exact pattern. It's the universal standard for unit testing.

## What Is Moq?

Moq is a .NET mocking library. It generates fake implementations of interfaces at runtime:

```csharp
var mock = new Mock<IUserRepository>();

// Configure: when GetByEmailAsync is called with any string, return fakeUser
mock.Setup(r => r.GetByEmailAsync(It.IsAny<string>(), It.IsAny<CancellationToken>()))
    .ReturnsAsync(fakeUser);

// Verify: confirm GetByEmailAsync was called exactly once
mock.Verify(r => r.GetByEmailAsync("test@example.com", default), Times.Once);
```

## Interview Questions & Answers

**Q: What is the difference between unit test and integration test?**

- **Unit test**: Tests ONE class in isolation. All dependencies mocked. No DB, no HTTP. Fast.
- **Integration test**: Tests multiple real layers working together. Involves real DB calls, file system, HTTP. Slower, but proves end-to-end correctness.

**Q: Why not just run the real code against a real test database?**

1. Slow — DB setup/teardown per test is expensive
2. Flaky — tests can fail due to DB state issues unrelated to the code
3. Hard to isolate — you can't test "what happens if GetByEmailAsync returns null" without inserting specific DB data

**Q: What does `It.IsAny<string>()` mean?**
A Moq matcher — "match any string argument". Without it, the mock only triggers when the exact value is passed. `It.Is<string>(s => s.Contains("@"))` would narrow to only emails.

**Q: What testing framework does this use?**
NUnit for test runner attributes (`[Fact]`). Moq for mocking. xUnit assertions (`Assert.Equal`, `Assert.ThrowsAsync`). The test project references both the `LearnSphereBackend` project and the NUnit/Moq NuGet packages.
